# Как обновлять базу менеджера — шпаргалка

Два скрипта, которые сами тянут данные с aprellshop.ru. Работают у тебя на Mac, токены не тратят.

- `update_photos.py` — подтягивает **фото** галерей.
- `update_prices.py` — подтягивает **актуальные цены** (текущую и старую зачёркнутую).

Оба лежат в одной папке с `data.json` и `index.html`. Ничего доустанавливать не нужно — только Python, который уже есть.

---

## Быстрый старт

Открой Терминал, перейди в папку с базой и запусти нужный скрипт:

```
python3 update_photos.py     # добрать фото у цветов без картинок
python3 update_prices.py     # сверить и обновить цены
```

После этого загрузи обновлённые `data.json` и `index.html` на GitHub, как обычно.

---

## Скрипт фото — `update_photos.py`

**Когда запускать:** появился новый цвет; на сайте обновили фотосессию.

**Что делает:** находит цвета, у которых есть ссылка на страницу товара, но нет фото, заходит на страницы и вписывает по 5 кадров галереи.

**Как «показать» скрипту цвет без фото:** никак специально — он сам берёт все цвета, где ссылка есть, а фото нет. Пустое поле фото = сигнал «добери».

**Новый цвет, которого ещё нет в базе:** добавь строку в блок `"site"` нужной модели в `data.json`:

```
{ "a": "S5114", "c": "Бордовый", "u": "https://aprellshop.ru/catalog/sumka/bolshie-sumki/s5114-55" }
```
Три поля: `a` — артикул варианта, `c` — цвет, `u` — ссылка на страницу. Фото не пиши — скрипт добавит сам.

**Режимы:**
```
python3 update_photos.py --refresh    # перекачать фото у ВСЕХ цветов заново
python3 update_photos.py --limit 8    # брать по 8 кадров вместо 5
```

## Скрипт цен — `update_prices.py`

**Когда запускать:** перед рассылкой; в начале и в конце распродажи; при любом сомнении в ценах.

**Что делает:** заходит на карточку каждого цвета и вписывает его цену отдельно — `price` (текущая) и `oldPrice` (старая зачёркнутая, если есть скидка). Когда скидка на сайте кончилась, старую цену убирает.

**Отчёт:** после прогона печатает в Терминал и сохраняет в `price_report.txt` список, что и где изменилось, со стрелками. Например:
```
0360 · Пекан: 19 900 ₽  →  13 990 ₽ (старая 19 900 ₽) ↓
```
Если ничего не поменялось — так и напишет. Что не смог прочитать — вынесет отдельным списком со ссылкой, чтобы глянуть вручную.

---

## Что нужно помнить

- **Ссылка есть → скрипт добудет всё сам.** Присылать мне страницы товаров не нужно — это тратит токены впустую (страница целиком ложится в диалог).
- **Скрипты не мешают друг другу:** фото трогает только `update_photos.py`, цены — только `update_prices.py`. Можно гонять по отдельности.
- **Откат:** перед сохранением оба кладут рядом старые версии — `data.backup.json` и `index.backup.html`. Если что-то пошло не так, вернись к ним.
- **Порядок при сомнениях:** сначала запусти скрипт, потом смотри отчёт/результат, потом грузи на GitHub.

## Если скрипт споткнулся

Если в выводе Терминала строка вида `! ... : ошибка` или `цена не распознана` — скопируй **вывод из Терминала** (не саму страницу сайта) и пришли мне. Поправлю шаблон. Такое может случиться, если на сайте поменяли вёрстку карточки.
